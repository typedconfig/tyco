Tyco typed configuration

Overview:

Tyco is a text-based configuration format which describes statically-typed tabular data. It aims to provide a low cognitive overhead for engineers through simplicity and de-duplication. Think of it as typed csv with globals, optional keys, and default values, rendered as something that looks like simplified yaml.

It's probably not for everyone. If you enjoy type-inference in your config attributes (and surprises at runtime) or ugly nested layouts, then you might want to stick with something else. But if you hardly sleep, and night after night you sit by your computer, looking for yet another markdown language, then Tyco may be worth a shot. And if you've come this far, maybe you're willing to come a little further (try it out!).

What Tyco offers:

  • Explicitly-typed configuration for attribute type guarantees.
  • Just enough flexibility to match the shape of the data.
  • Supports references to other objects for internal consistency.
  • Can set defaults in multiple places to keep things DRY.
  • Clean syntax for engineers who care about aesthetics.
str timezone: UTC     # this is a global config setting

Application:
  str service:
  str profile:
  str command: start_app {service}.{profile} -p {port.number}
  Host host:
  Port port: Port(http_web)
  - service: webserver, profile: primary, host: Host(prod-01-us)
  - service: webserver, profile: backup,  host: Host(prod-02-us)
  - service: database,  profile: mysql,   host: Host(prod-02-us), port: Port(http_mysql)
                      
Host:
 *str hostname:  # star character (*) indicates primary key for lookups
  int cores:
  bool hyperthreaded: true
  str os: Debian
  - prod-01-us, cores: 64, hyperthreaded: false
  - prod-02-us, cores: 32, os: Fedora

Port:
 *str name:
  int number:
  - http_web,   80
  - http_mysql, 3306

Tyco prioritizes readability

Tyco's syntax is designed for humans to read, write, and review. It emphasizes clarity and simplicity, making config data easy to understand at a glance.

  • Inline schema defined once, used everywhere.
  • No functions, just objects.
  • Comments and #include files are supported.
  • Keyed access to structured data.
  • Light-weight {templating} to avoid repetition.
  • Combines the best of toml, yaml, json, python and others.

Tyco has explicit types

Tyco includes first-class, well-defined primitives so applications can rely on consistent values at runtime. Footguns kept to a minimum.

  • Base types: str, int, bool, float, decimal, date+time.
  • Custom structs: composed of base types (or other structs).
  • Can reference other objects based on the primary key (*).
  • Optional: flags indicating array or nullable.
  • Global scope config variables supported.

Tyco is easy to adopt

Tyco has tooling to help migrate an existing configuration.

  • Can both load/dump from several well-known serialization formats.
  • Subclassing structs in python for additional validators.
  • Syntax highlighting available in editors and a CLI application.
  • Parsers are being developed in all major languages.
